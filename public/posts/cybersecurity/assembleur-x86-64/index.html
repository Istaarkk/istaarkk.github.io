Assembleur x86-64 : Registres, Conventions d'Appel et Exemples

Guide complet sur l'assembleur x86-64, ses registres et son fonctionnement pour la programmation bas niveau

Apr 4, 2024
5 minute read

L’assembleur est souvent considéré comme le pont entre les langages de haut niveau et le code machine brut. Comprendre l’assembleur x86-64 est essentiel pour le reverse engineering, l’exploitation de binaires et l’optimisation de code.

1. Registres x86-64

L’architecture x86-64 dispose de 16 registres généraux de 64 bits :

  • RAX : Accumulateur principal, souvent utilisé pour les opérations arithmétiques ou pour retourner une valeur (comme return en C).
  • RBX : Registre de base, souvent préservé.
  • RCX : Registre de compteur, utilisé pour les boucles (par exemple avec loop).
  • RDX : Registre de données, utilisé pour les opérations I/O ou pour passer un second argument dans les appels de fonction.
  • RSI : Index source, utilisé pour les opérations sur les chaînes ou pour passer le premier argument dans les appels de fonction.
  • RDI : Index destination, utilisé pour les opérations sur les chaînes ou pour passer le deuxième argument dans les appels de fonction.
  • RBP : Pointeur de base, pointe vers le début de la pile de la fonction courante.
  • RSP : Pointeur de pile, pointe vers le sommet de la pile, généralement utilisé pour les opérations liées à la pile.
  • R8 à R15 : Registres généraux supplémentaires.

Il existe également des versions plus petites des registres pour 32 bits, 16 bits et 8 bits :

  • RAX → EAX (32 bits) → AX (16 bits) → AL (8 bits)

2. Convention d’appel : SysV ABI pour Linux

En x86-64, la convention d’appel SysV ABI est utilisée pour passer des arguments lors des appels de fonction.

  • Les six premiers arguments sont passés dans les registres suivants, de gauche à droite :

    • 1er argument → RDI
    • 2ème argument → RSI
    • 3ème argument → RDX
    • 4ème argument → RCX
    • 5ème argument → R8
    • 6ème argument → R9

  • Les valeurs de retour sont stockées dans RAX.

  • Les arguments supplémentaires sont passés via la pile.

  • Le registre RSP doit être aligné sur 16 octets avant tout appel de fonction.

3. Exemples comparant C et NASM

Exemple 1 : Addition de deux nombres

En C : 

int add(int a, int b) {
    return a + b;
}

En NASM : 

section .text
    global add    ; Déclarer la fonction comme publique

add:
    mov rax, rdi  ; Copier le premier argument (a) dans RAX
    add rax, rsi  ; Ajouter le second argument (b) à RAX
    ret           ; Retourner le résultat dans RAX

Explications :

  • En C, les arguments a et b sont passés à la fonction, et la somme est retournée.
  • En assembleur, rdi contient le premier argument a et rsi contient le second argument b. Nous effectuons l’addition et retournons le résultat dans rax.

Exemple 2 : Boucle for

En C : 

void loop(int n) {
    for (int i = 0; i < n; i++) {
        // faire quelque chose
    }
}

En NASM : 

section .text
    global loop

loop:
    mov rcx, rdi     ; Copier n dans RCX (RCX utilisé comme compteur)
    xor rax, rax     ; Mettre i à 0 (utilisant RAX comme i)

for_loop:
    cmp rax, rcx     ; Comparer i (RAX) avec n (RCX)
    jge end_loop     ; Si i >= n, sauter à end_loop
    ; Faire quelque chose ici

    inc rax          ; i++
    jmp for_loop     ; Répéter la boucle

end_loop:
    ret              ; Retourner

4. Gestion de la mémoire et de la pile

La pile est une structure LIFO (Last In, First Out). Chaque fonction peut avoir son propre espace sur la pile pour stocker des variables locales. Lorsqu’une fonction est appelée, un nouveau cadre de pile est créé.

  • PUSH : Ajoute une valeur au sommet de la pile (décrémente rsp).
  • POP : Retire une valeur du sommet de la pile (incrémente rsp).

Exemple : Fonction avec une variable locale

En C : 

void foo() {
    int x = 10;
    // faire quelque chose avec x
}

En NASM : 

section .text
    global foo

foo:
    push rbp         ; Sauvegarder l'ancien RBP
    mov rbp, rsp     ; RBP pointe maintenant vers le sommet actuel de la pile

    sub rsp, 8       ; Allouer de l'espace pour x (8 octets)
    mov qword [rbp-8], 10  ; Initialiser x avec la valeur 10

    ; Faire quelque chose avec x

    mov rsp, rbp     ; Restaurer la pile
    pop rbp          ; Restaurer l'ancien RBP
    ret

5. Appels système en x86-64 Linux

Pour effectuer un appel système en assembleur (comme lire un fichier, imprimer à l’écran, etc.), nous utilisons l’instruction syscall.

  • Les appels système sont identifiés par un numéro, qui est passé dans RAX.
  • Les paramètres sont passés dans les registres suivants : RDI, RSI, RDX, R10, R8, R9.

Exemple : Appel système pour écrire sur la sortie standard

En NASM : 

section .data
    msg db "Hello, world!", 0xA   ; Message à afficher
    len equ $ - msg                ; Longueur du message

section .text
    global _start

_start:
    ; Appel système write (numéro 1)
    mov rax, 1        ; Numéro d'appel système (write)
    mov rdi, 1        ; Descripteur de fichier (1 pour stdout)
    mov rsi, msg      ; Adresse du message
    mov rdx, len      ; Longueur du message
    syscall           ; Effectuer l'appel système

    ; Appel système exit (numéro 60)
    mov rax, 60       ; Numéro d'appel système (exit)
    xor rdi, rdi      ; Code de retour (0)
    syscall           ; Quitter

6. Application en sécurité informatique

L’assembleur est un outil essentiel en sécurité informatique pour plusieurs raisons :

  1. Reverse Engineering : Comprendre comment un programme ou un malware fonctionne en analysant son code désassemblé
  2. Exploitation de vulnérabilités : Créer des shellcodes et des exploits qui nécessitent une manipulation précise de la mémoire
  3. Analyse de binaires : Identifier des vulnérabilités et des comportements suspects dans des exécutables
  4. Forensique : Analyser des processus malveillants directement au niveau du code machine

La maîtrise de l’assembleur x86-64 est donc un atout majeur pour tout spécialiste en cybersécurité, particulièrement dans le domaine de la sécurité offensive et de l’analyse de malware.
