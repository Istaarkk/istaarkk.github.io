Web Fuzzing - Guide Complet

Notes détaillées sur le fuzzing web, basées sur le cours HTB

Apr 4, 2024
2 minute read

Présentation

Ces notes couvrent le cours Web Fuzzing sur Hack The Box Academy. Le fuzzing est une technique essentielle pour découvrir des ressources cachées sur un site web.

Fuzzing de base

Pour trouver des répertoires cachés, on peut utiliser Feroxbuster, un outil open-source puissant :

feroxbuster -u http://exemple.com/FUZZ -w /chemin/vers/wordlist.txt -x php,html,txt

Fuzzing récursif

Pour le fuzzing récursif (explorer les sous-répertoires automatiquement) :

feroxbuster -u http://exemple.com/recursive_fuzz -w /chemin/vers/wordlist.txt -x php,html,txt

Fuzzing de paramètres avec Gobuster ou FFUF

Pour fuzzer les paramètres d’une URL, FFUF est particulièrement efficace :

ffuf -c -ic -t 200 -u http://exemple.com/get.php?x=FUZZ -w /chemin/vers/common.txt -mc 200

Fuzzing de Vhost (Virtual Hosts)

Pour découvrir des sous-domaines hébergés sur le même serveur :

  1. Ajoutez d’abord l’hôte dans votre fichier hosts :
echo 'IP.du.serveur domaine.exemple' | sudo tee -a /etc/hosts
  1. Puis utilisez FFUF pour le fuzzing :
ffuf -c -ic -t 200 -u http://IP.du.serveur -H "Host: FUZZ.domaine.exemple" -w /chemin/vers/subdomains.txt

Fuzzing basé sur la taille de la réponse

Pour trouver des répertoires cachés en se basant sur la taille de la réponse :

feroxbuster -u http://exemple.com -w /chemin/vers/wordlist.txt -x .php .html .gz -t 300

Pour vérifier la taille exacte d’un fichier :

curl -I http://exemple.com/chemin/vers/fichier.gz

Fuzzing d’API

Pour découvrir des endpoints d’API :

ffuf -c -ic -t 200 -w /chemin/vers/endpoints.txt -u http://api.exemple.com/FUZZ -mc 200

Combinaison de techniques

Pour les scénarios complexes, une approche en plusieurs étapes est souvent nécessaire :

  1. Découvrir les répertoires avec feroxbuster
  2. Trouver les fichiers sensibles avec ffuf
  3. Exploiter les paramètres avec des payloads spécifiques

Conseils pratiques

  • Utilisez le filtre -fs pour ignorer les réponses d’une certaine taille
  • Augmentez les threads -t pour accélérer le fuzzing
  • Utilisez -mc pour filtrer par code de statut HTTP
  • Combinez avec -v pour obtenir plus de détails sur les requêtes

Le fuzzing est une compétence indispensable en pentest web, permettant de découvrir des ressources qui ne sont pas directement accessibles via la navigation normale.
